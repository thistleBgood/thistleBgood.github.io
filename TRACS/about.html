TRACS - Teaching Resource for Annotating Code Samples

Using TRACS Online

Step 1 - Import File

Upload the file you want to annotate by clicking on the "Import" button and navigating to the location of your file. TRACS will automatically add some range annotations to the file before displaying it on screen.

Step 2 - Range annotations

TRACS assumes that every line is an individual statement. Although this is the case most of the time, sometimes there are more than one per line, and sometimes they span across multiple lines. The contextual menu will allow you to make changes as appropriate.

Once the main structure of the statements is in place, you can begin to add expression ranges to the code. To do this, just highlight the code and click "Annotate". You can nest both statements and expressions inside one another. If you make a mistake, you can also switch which type of range it is by clicking the contextual menu.

Step 3 - Static Control Flow

To add static control flow arrows, click and drag from one range to another. To add decision points, click and drag from a range that already has an arrow coming from it.

Step 4 - Dynamic Control Flow

The dynamic control flow layer consists of three parts:

  • A trace table in the top middle section;
  • An evaluator in the bottom middle section;
  • And a files section in the right hand section.

Each section starts out empty. Files starts out with three default file locations: stdin, stdout and stderr. If you don't need the files, or you want to add more, you can do this through the contextual menu.

There is also a step counter, which allows you to move forwards or backwards through the execution of the program.

To add an entry to any of the three sections, click on the + button. Fill in the value and it will automatically be marked with the current step. To modify a previously entered value, just click on the pop up contextual menu.

You can also have multiple dynamic control flow layers, each with different values and files.
